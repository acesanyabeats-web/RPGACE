RPGACE — MASTER EXECUTION PLAN (Fable 5, July 17 2026)
Response to fable_handoff_backlog.txt. This is the step-by-step plan Alex
asked for: each SESSION below is one sitting with ONE goal, an exact
paste-able prompt to start it, and a hard pass/fail line so it's obvious
when it's finished. Do them in order unless a session's own text says it
can float.

THE ONE STRUCTURAL RULE THIS PLAN ENFORCES
The backlog's honest diagnosis was: sessions got slow because many goals
ran in one sitting and bug reports didn't name their entry point. So:
one session = one goal = one pass/fail. If something unrelated breaks
mid-session, it gets a one-line note added to this file's PARKING LOT
(bottom) and the session stays on its goal.

BUG-REPORT TEMPLATE (use every time — this alone saves the most rounds)
  1. Which button/entry point, exactly (e.g. "✍️ paste TOC", "📖 Start
     URL", "📎 PDF upload", chapter-text box)
  2. What you typed/pasted/selected (the actual URL, file name, or text)
  3. What you expected vs what appeared (screenshot or paste the screen)
That's it. With those 3 lines, diagnosis starts at the right code path
immediately instead of 3 rounds in.

ARCHITECTURE HARD LINES (never violate without explicit confirmation)
1. main.js is FROZEN. Edits only as deliberate, logged exceptions,
   with `node --check main.js` before commit + a patch_notes card the
   same session.
2. All new code goes in rpgace_core.js via RPGACE.register(), between
   /* ===MODULE:x=== */ ... /* ===END:x=== */ markers. Run
   `node --check rpgace_core.js` after EVERY change.
3. Never add a third static <script> tag to index.html (breaks the
   password gate via race conditions). Runtime-injected scripts via
   document.createElement('script') are the approved pattern.
4. Never Python-string-replace on index.html (exportEncyclopedia
   contains a fake </body></html> inside a JS string). Targeted edits
   only.
5. All fixed overlays are createElement'd and appended to
   document.body — never inside .page divs.
6. Every id="page-X" div lives inside #app, before the script tags.

CODE LANDMINES (each one has already caused a real bug)
7. RPGACE.sb.insert() does NOT return the inserted row. If you need
   the new row's id, use raw fetch + 'Prefer: return=representation'
   + a response.ok check (pattern: _insertNewSteps,
   _createBookFromExtraction). This exact mistake caused the
   "Chapter 0 of 0" silent failure.
8. hooks.fire() never revisits listeners added mid-fire — any module
   calling hooks.on('rpgace:ready', ...) inside its own init()
   silently never fires. When in doubt, call the function directly;
   'rpgace:ready' has always already fired by the time init() runs.
9. Writes done via raw fetch bypass RPGACE.cache — call
   RPGACE.cache.clear('tablename') after them or reads are stale up
   to 60s.
10. Word-boundary regex treats hyphens as boundaries (bare "roll"
    matches inside "b-roll") — keyword lists use compound phrases,
    never bare adjectives.

PROCESS RULES (non-negotiable, all learned the hard way)
11. Pull real source before editing. Never guess at function
    internals — Read/Grep the actual current code first.
12. One failed fix = STOP. Get real evidence before a second attempt.
    The strongest form: query Supabase directly (source_url, row
    counts, null columns) to confirm which code path actually
    produced the data — the recurring "TOC saved as chapter 1" pain
    was Alex unknowingly using a different entry point than the one
    being fixed.
13. Grep before adding any new UI behavior — this project has a
    repeated history of duplicate implementations.
14. Multi-subsystem asks (3+ new pieces) get spec'd and confirmed
    BEFORE code. Record Alex's answers VERBATIM in a committed .txt
    file (pattern: bookworm_spec_backlog.txt). Never substitute your
    recommended defaults for his actual answers — doing so once
    genuinely angered him, and rightly.
15. Real evidence before positioning/layout code — screenshot or
    real DOM output, never inferred geometry.
16. Fail loud: surface errors as toasts, never silently swallow a
    failed write or fake success/progress.
17. Every taxonomy write gets a human checkpoint before committing
    (inline confirm popup, or staging via taxonomy_proposals). AI
    proposes, human confirms — no exceptions, this has caught real
    garbage twice.
18. Never trust model JSON blindly — parse defensively, sanitize
    structure-affecting output in code (pattern: _sanitizeNewSteps).
19. Long AI jobs stream: await only the first result, continue in
    background writing to Supabase, poll from the UI. Never make
    Alex wait minutes staring at nothing.
20. Destructive actions get two-click arm/confirm (pattern:
    Bookworm's 🗑 with 3s timeout).
21. Chat triggers: chainable window.sendChat wrap with a
    TRIGGER_PREFIXES array + a window._xPatched guard flag; always
    fall through to the original for non-matching input.
22. Mobile-first: every new control needs a thumb-sized target.
23. New cross-module signals go through RPGACE.hooks named events —
    never a second MutationObserver on the same DOM node.

MODEL IDs (exact, no guessing)
