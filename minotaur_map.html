🐂The Minotaur Map

A mental-map reading of RPGACE's information architecture, in the shape of a labyrinth. The Minotaur is the information itself — a real user input (a chat message, a book, a URL, a shift) — wandering the maze from an entrance door, through chambers where it is read, judged, and transformed, to the one exit where it finally re-emerges as something Alex actually sees on screen. Every wall is a real module boundary; every fork is a real yes/no decision RPGACE's code actually makes.

🚪 Entrance — where a real input enters the maze
🔀 Fork — a real yes/no decision in the code
🏛️ Chamber — where information is processed or stored
🚧 Checkpoint — a human (Alex) must pass the Minotaur through
🏆 Exit — where the Minotaur finally reaches daylight: rendered on screen

Reading this map honestly: this is a mental-map-level representation of RPGACE's primary information corridors — the same scope system_flow_map.md (the 5th oversight doc) covers in Mermaid-diagram form, redrawn here as a single spatial metaphor rather than separate per-pipeline diagrams. It intentionally does not attempt to draw every sub-corridor (Fusion Links' cross-phylum search, Concept Fusion's branch-merging, the ~25 unaudited hooks.on('rpgace:ready') sites) — those live in system_flow_map.md's dedicated diagrams and the built/unverified/not-built truth table. Update this map whenever a NEW wing is added (a new entrance, a new hub, a new exit) — not for every internal patch inside an existing corridor, which is what patch_notes.html is for.
