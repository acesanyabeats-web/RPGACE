🐂The Minotaur Map

system_flow_map.md is written for the nerds — exact function names, exact Mermaid syntax, exact code terms. This page is its plain-language twin: the Minotaur is a real piece of information — a chat message, a book chapter, an idea worth teaching — carried like water down a river system. Tributaries are the places information enters. Rivers are the paths it's carried along. Estuaries are where several rivers meet and become one shared current. The sea is where it finally reaches Alex's screen. Every river below is a direct plain-language translation of one real Mermaid diagram — nothing here was invented separately from it, so the two documents can never quietly drift apart. Restructured July 23 so the ten rivers now read in real chronological order — the actual step-by-step journey a piece of information takes, start to finish, not the order each piece was originally built in.

Shape key (matches Mermaid's own convention exactly)
Spring / river mouth — where information enters or finally exits
Chamber — a real place along the river where information is read, judged, or changed
Fork — a real yes/no decision the code actually makes
Reservoir — a Supabase table, where information is stored and can be drawn from later
Dry riverbed — planned, not yet dug (not built)
Metaphor key
🐂 The Minotaur — the actual information travelling the system right now
🌊 Tributary — one specific input source feeding a river
🌀 Estuary — where multiple rivers converge into one shared current
🏖️ The Sea — the final output Alex actually sees
The whole system, zoomed out
