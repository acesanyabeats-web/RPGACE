The Governing Rule
"Does this feature directly result in a beat being made or a video posted within 48 hours of using it?"

Yes → build immediately. No → wait. Standing exception: RPGACE infrastructure work is exempt.

Architecture
Three Layers + Two Real Hubs
LAYER 1: main.js (FROZEN) — core rendering, chat, calendar, encyclopedia
LAYER 2: rpgace_core.js — all domain modules, via RPGACE.register()
LAYER 3: api/ (Vercel serverless) — oracle.js, intel.js, composio.js

The two real hubs everything converges on (confirmed via full interconnection audit): Oracle (Claude API, generation) and Taxonomy Tree (knowledge structure). Only the Schedule system runs fully independent of both — shifts, agendas, and calendar form a self-contained loop with no taxonomy dependency.

  1. py rpgace_build.py new DOMAIN moduleName
  2. Write logic in mod_moduleName.js
  3. py rpgace_build.py add DOMAIN mod_moduleName.js
  4. py rpgace_build.py check
  5. git add rpgace_core.js && git commit -m "..." && git push

Currently Open Bugs
Oracle 504 timeout on long structured responses
maxDuration raised to 60s — insufficient. Response-shape-dependent: long 3-layer teaching format times out, shorter Socratic-method mode completes fine on identical topic. July 15: reproduced concretely in Phylum Path's article-generation call (max_tokens:1800 failed outright with a JSON-parse error) — confirmed the failure scales with requested length (700 tokens worked, 1200 got truncated, 1800 failed). Mitigated for those 2 calls via trimmed prompts + lower token ceilings, but the underlying timeout is untouched — needs streaming or chunked generation, not another blind trim.
~25 places in rpgace_core.js may share the rpgace:ready re-subscription bug
Found July 16 diagnosing the Phase 2 nav tab: RPGACE.register() wires every module's init() as a listener on 'rpgace:ready' itself, so a module calling RPGACE.hooks.on('rpgace:ready', ...) from inside its own init() registers too late — RPGACE.hooks.fire()'s plain Array.forEach never revisits listeners pushed during its own iteration, so that listener can silently never fire. phylumPath's 2 occurrences fixed (call the injector directly instead of re-subscribing); ~25 more occurrences of the same call elsewhere in the file are flagged but not individually audited.

Standing Development Rules
  1. Screenshot/real-source before writing layout or DOM code — never infer.
  2. Grep for existing functions before adding new UI behaviour.
  3. Prefer normal document flow over absolute positioning.
  4. One failed fix = stop, get real evidence, don't patch blind again.
  5. Multi-subsystem requests get spec'd and deferred whole, never half-built.
  6. Every doc update applies to all three docs by default (confirmed July 8).
