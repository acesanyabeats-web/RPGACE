The Architecture Decision

"Does this feature directly result in a beat being made or a video posted within 48 hours of using it?" — The governing rule. Every architectural decision answers to this.

The Root Problem — Diagnosed

Every bug this session had the same cause: Python scripts doing fragile string-matching on a monolithic JS file. The file grows, patterns don't match, modules duplicate, things break. The fix is not another patch script. It is a permanent structural change.

The Three-Layer Model

┌─────────────────────────────────────────────────────────────┐
│  LAYER 1: FOUNDATION (FROZEN after tested)                  │
│  rpgace_core.js — STATE · hooks · DB · utils · bridge      │

Module Anatomy — The Standard

/* ===MODULE:moduleName=== */
RPGACE.register('moduleName', {

  init: function() {
    var self = this;
    /* 1. Wire page:show hook */
    RPGACE.hooks.on('page:show', function(pageName) {
      if (pageName === RPGACE.CONFIG.pages.oracle) {
        setTimeout(function() { self._setup(); }, 400);
      }
    });
    /* 2. Wire rpgace:ready */
    RPGACE.hooks.on('rpgace:ready', function() {
      setTimeout(function() { self._setup(); }, 500);
    });
    /* 3. Intercept main.js functions if needed */
    /* window.toggleProdOraclePanel = function() { self.open(); }; */
  },

  _setup: function() { /* inject DOM, patch functions */ },
  _supabaseDel: function(table, filter) { return RPGACE.sb.del(table, filter); },
  _supabaseInsert: function(table, row)  { return RPGACE.sb.insert(table, row); },

});
/* ===END:moduleName=== */

RPGACE.CONFIG — The Binding Layer

RPGACE.CONFIG.pages.research     → 'learning'   (fixes the mismatch bug permanently)
RPGACE.CONFIG.mainFns.prodOracle → 'toggleProdOraclePanel'
RPGACE.CONFIG.supabase.url       → 'https://gripopghczmrbrhqtqbm.supabase.co'
RPGACE.sb.del('intel_reports', 'id=eq.UUID')    → delete from Supabase
RPGACE.sb.insert('intel_bibliography', {...})   → insert into Supabase

Any module that needs Supabase, page names, or main.js function names references CONFIG. When anything changes, you edit one line.

RPGACE.hooks — The Event Bus

'rpgace:ready'     → Fires 150ms after window.load. All modules initialized.
'page:show'        → Fires when user navigates tabs. Payload = page name from CONFIG.
'xp:awarded'       → Fires when XP is added. Any module can respond.
'journal:saved'    → Fires when a journal entry is saved.
'state:change'     → Fires when STATE changes. Payload = key, value.

The Only Commands You Need

Add a new feature

# 1. Create a module stub
python rpgace_build.py new CONTENT beatLog

# 2. Implement mod_beatLog.js in any text editor

# 3. Add to the correct domain
python rpgace_build.py add CONTENT mod_beatLog.js

# 4. Deploy
python rpgace_build.py deploy "Step 12: Beat Log"

Update an existing module

# Edit mod_instaOracle.js
# Then:
python rpgace_build.py update instaOracle mod_instaOracle.js
python rpgace_build.py deploy "Update: instaOracle new commands"

Check state of the file

python rpgace_build.py list      # See all modules by domain
python rpgace_build.py check     # Syntax + duplicates
python rpgace_build.py status    # Overview + recent git log

Remove a broken module

python rpgace_build.py remove brokenModuleName
python rpgace_build.py deploy "Remove: brokenModuleName"
⚠ NEVER DO THIS AGAIN: Writing JavaScript as Python string literals. Every module lives in its own .js file. Python only handles file operations, never JS content.
